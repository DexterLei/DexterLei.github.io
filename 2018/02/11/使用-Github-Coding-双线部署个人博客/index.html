使用 Github&Coding 双线部署个人博客

发表于 2018-02-11 | 分类于 博客 ， Hexo ， Next | Views
字数统计: 734 字 | 阅读时长 ≈ 3 分钟

写在前面

考虑到潜在 Github 无法访问的风险，以及有利于百度 SEO，遂将个人博客同时部署到 Coding 和 Github （的 Page ）上。当从国内访问我的博客时，域名指向 Coding，反之指向 Github。

Coding 设置

创建静态 Coding Pages

新建一个名为 {user_name}.coding.me 的项目。{user_name} 您本人的用户名（非昵称），使用其他人的用户名创建的项目不会被归为『用户 Pages』类型。​

注意：Coding Pages 成功部署后您的网站可被公开访问，私有项目也是如此。

在参考资料中看到设置为私有可能有些问题，考虑到都是会被公开，索性就设置为公开。

创建完毕后网页会自动跳转到新建的项目位置，选择 Pages 服务，设置部署来源为 Master 分支。

保存后会出现自定义域名选项，但是需要完善用户信息，在此不赘述，完善即可，自定义域名暂时无需绑定。

SSH Keys 设置

打开 Coding 个人设置页面，选择侧边栏中的 SSH 公钥，将之前生成的公钥内容（id_rsa.pub 文件）粘贴到选框中，密码仍旧是公钥生成时的密码，在推送时会需要输入。
